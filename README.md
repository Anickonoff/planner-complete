# Planner

Планировщик задач для 2-3 человек.
Приложение состоит из:

- `backend` - Node.js/Express API, Telegram-планировщик задач для группового чата с напоминаниями, дедупликацией и управлением событиями через команды и API для связи с frontend.
- `frontend` - React/Vite интерфейс c фунциями создания, редактирования, удаления задач.

Проект рассчитан на **один общий чат** (семейный / командный), где несколько пользователей могут добавлять и управлять событиями.
Более подробно об параметрах серверного компонента можно прочитать в backend/readme.md

---

## Возможности

### 📌 Работа с событиями

- создание событий из текста;
- валидация (нельзя создавать события в прошлом);
- хранение в JSON (UTC);
- резервное копирование перед каждой записью.

### ⏰ Напоминания

- ежедневные напоминания о событиях на завтра;
- еженедельный список событий;
- дедупликация (одно напоминание — один раз);
- устойчивость к перезапускам.

### 📋 Команды Telegram

- `/help` — справка
- `/status` — количество событий и ближайшее
- `/today` — события на сегодня
- `/tomorrow` — события на завтра
- `/week` — события на следующую неделю
- `/delete <id>` — запрос удаления
- `/confirm <id>` — подтверждение удаления
- `/cancel` — отмена удаления

### 🌍 Часовые пояса

- хранение дат в UTC;
- отображение и сравнение в заданном TZ (например `Asia/Tomsk`);
- корректная локаль (`ru-RU`).

### 🛠 Надёжность

- long polling Telegram (без webhook);
- персистентный offset;
- дедупликация напоминаний;
- graceful shutdown;
- файловое логирование.

## Формат ввода событий

```
ДД.ММ [ЧЧ:ММ] Название [Описание]
```

Примеры:

```
12.01 Работа
1.2 Работа
12.01 8:30 Работа
15.01 18:30 Встреча с командой
```

Поддерживаются форматы времени:

- `8:30`
- `08:30`

Дата может быть указана как с ведущими нулями, так и без них:

- `1.2`
- `01.02`
- `7.1`
- `07.01`

## Quick Start

### Скачивание из git

```bash
git clone <repo-url>
cd planner-complete
```

### Docker

Для запуска через docker испльзуйте команду

```bash
docker compose up --build -d
```

После запуска в браузере откройте:

```text
http://localhost:3000
```

### Без Docker

Без docker можно запустить компоненты отдельно. В первом терминале для запуска серверной части с telegram-ботом выполните команды

```bash
cd backend
npm install
npm start
```

Во втором терминале для доступа через браузер выполните:

```bash
cd frontend
npm install
npm run dev
```

Откройте адрес Vite, обычно:

```text
http://localhost:5173
```

## Настройка Telegram

Скопируйте файл `backend/.env.example` в `backend/.env` и отредактируйте содержимое:

### Основные переменные

```env
TG_BOT_TOKEN=
TG_POLLING_INTERVAL=2000
TELEGRAM_ALLOWED_CHAT_ID=
TG_PROXY_URL=
```

### Пример `backend/.env`

```env
PORT=3000
DATA_FILE_PATH=./data/events.json
TG_BOT_TOKEN=
TG_POLLING_INTERVAL=2000
TELEGRAM_ALLOWED_CHAT_ID=
TG_PROXY_URL=
LOG_DIR=./logs
LOG_LEVEL=info
BACKUP_DIR=./data/backups
BACKUP_KEEP_DAYS=7
APP_TIMEZONE=Asia/Tomsk
APP_LOCALE=ru-RU
```

Если `TG_BOT_TOKEN` не задан, backend работает без Telegram.

`TELEGRAM_ALLOWED_CHAT_ID` — ID группового чата, где бот разрешён.

## Данные

- `backend/data/events.json` - основные данные
- `backend/data/backups/` - резервные копии
- `backend/logs/` - логи

При Docker Compose эти папки монтируются как volume.
